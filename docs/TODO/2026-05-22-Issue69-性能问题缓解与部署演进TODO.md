## Phase 3：gevent 单 worker 评估

**目标**：在不立刻拆 scheduler 的前提下，评估部署层最小缓解方案。

### Task 3.1：形成 gevent 切换前置条件
- [x] 明确需新增依赖：`gevent` pip 包
- [x] Dockerfile 变更：`-k gevent` + monkey patching
- [x] 受影响接口类型已列出（SSE / REST / 外网 / 调度器）

### Task 3.2：兼容性风险清单
- [x] APScheduler BackgroundScheduler：🔴 高风险（线程 → greenlet 替换）
- [x] ThreadPoolExecutor：🟡 中风险（`as_completed` 可能失序）
- [x] threading.Lock × 3：🟡 中风险（替换为 BoundedSemaphore）
- [x] SQLite WAL 共享：🟡 中风险
- [x] SSE/yield 流式：🟢 低风险
- [x] urllib.request：🟢 低风险

### Task 3.3：验证方案设计
- [x] 8 项最小验证矩阵
- [x] 3+4 项必过回归清单
- [x] 建议本地 Docker 验证环境

### Task 3.4：结果决策
- [x] **建议跳过 gevent**，直接进入 Phase 4 scheduler 拆分
- [x] 理由：Phase 1+2 已有效降低首屏负载；gevent 引入的调度器风险 > 预期收益
- [x] 若仍需尝试：最低限度使用策略 B + 48h staging 观察
